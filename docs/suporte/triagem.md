# Triagem de atendimento

Use este roteiro no primeiro contato com o usuário.

## Perguntas iniciais

| Pergunta | Objetivo |
| --- | --- |
| Qual módulo e tela você estava usando? | Localizar a rotina afetada |
| O que você estava tentando fazer? | Entender o objetivo do usuário |
| Qual mensagem apareceu? | Identificar erro conhecido ou validação |
| Acontece com todos os usuários ou só com um? | Separar permissão, cadastro ou falha geral |
| Acontece com todos os registros ou só com um cliente/produto/pedido? | Identificar problema de dado específico |
| Quando começou? | Relacionar com alteração, atualização ou configuração |

## Classificação do atendimento

| Tipo | Como identificar | Ação recomendada |
| --- | --- | --- |
| Dúvida de uso | Usuário não conhece a rotina | Enviar página do guia ou orientar passo a passo |
| Erro de cadastro | Falta dado obrigatório ou informação está inconsistente | Corrigir cadastro ou orientar responsável |
| Permissão | Usuário não acessa tela, botão ou operação | Validar grupo de usuário e perfil |
| Configuração | Regra do sistema não está parametrizada | Encaminhar para responsável pela configuração |
| Bug | Sistema apresenta comportamento inesperado e reproduzível | Coletar evidências e encaminhar para desenvolvimento |

## Checklist rápido

- [ ] Módulo identificado.
- [ ] Tela ou caminho informado.
- [ ] Usuário afetado registrado.
- [ ] Mensagem de erro copiada ou print anexado.
- [ ] Registro afetado informado, quando houver.
- [ ] Cenário reproduzido ou tentativa registrada.
- [ ] Solução ou encaminhamento documentado.
